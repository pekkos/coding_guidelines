Golden rule

Enforce these agreed upon guidelines at all times..

Every line of code should appear to be written by a single person, no matter the number of contributors.

Editor preferences

Set your editor to the following settings to avoid common code inconsistencies and dirty diffs:

  • Use soft-tabs set to two spaces.
  • Trim trailing white space on save.
  • Set encoding to UTF-8.
  • Add new line at end of files.

These preferences are documented in the .editorconfig file.

Learn more about EditorConfig and download a plugin for your text editor.

# EditorConfig: http://EditorConfig.org

[*]
indent_style = space
indent_size = 2
end_of_line = lf
charset = utf-8
trim_trailing_whitespace = true
insert_final_newline = true

Required tools and apps

The front end work is done locally outside of the Visual Studio environment, so make sure you have the required tools and apps up and running.

Web server

You need a local Apache web server with PHP to host your development files. Try one of these: XAMPP, MAMP or AMPPS.

Text editor

Your text editor of choice. A few recommendations:

  • Sublime Text (Mac and Windows)
  • TextMate (Mac only)
  • Atom (Mac and Windows)

Sass compiler

You need a good Sass compiler app. Go with CodeKit (Mac only) or PrePros. There is also the choice of Koala, but it lacks JS concatenating and such.

Pattern library

The front end environment consists of a Pattern Lab instance (http://patternlab.io/), where development is made in the Source folder, while the Public folder contains a generated static site that showcases various partials and templates that are being developed. Pattern Lab is a tool built around the principles of Atomic Design.

Atomic Design

Atomic design provides a clear methodology for crafting design systems. Clients and team members are able to better appreciate the concept of design systems by actually seeing the steps laid out in front of them.

Atoms

Atoms are the basic building blocks of matter. Applied to web interfaces, atoms are our HTML tags, such as a form label, an input or a button.

Molecules

Molecules are groups of atoms bonded together to build up a function, such as a search function containing a label, a text input field and a button.

Organisms

Molecules give us some building blocks to work with, and we can now combine them together to form organisms. A header would be an organism, where the search function would be on eof it's molecules.

Templates

Templates consist mostly of groups of organisms stitched together to form pages.

Pages

Pages are specific instances of templates. Here, placeholder content is replaced with real representative content to give an accurate depiction of what a user will ultimately see.
